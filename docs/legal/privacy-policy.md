## 1. 수집하는 개인정보 항목

Connect는 서비스 제공을 위해 다음과 같은 개인정보를 수집합니다.

### 1.1 회원가입 시 수집 정보 (필수)

**소셜 로그인 (OAuth 2.0) 연동 정보**
- **카카오 로그인**: 이름, 이메일 주소, 프로필 이미지 URL
- **네이버 로그인**: 이름, 이메일 주소, 프로필 이미지 URL

**법적 근거**: 개인정보 보호법 제15조 제1항 제1호 (정보주체의 동의)

### 1.2 조직 프로필 작성 시 수집 정보 (필수)

**조직 기본 정보**
- 조직명, 조직 유형 (기업/연구기관), 사업자등록번호, 대표자명, 설립연도

**조직 상세 정보**
- 주소 (시/도, 시/군/구, 상세 주소), 웹사이트 URL, 연락처, 이메일

**연구개발 정보**
- 산업 분야, 기술 분야, 연구개발 단계 (TRL), 연구개발 수행 여부, 주요 연구 성과

**법적 근거**: 개인정보 보호법 제15조 제1항 제1호 (정보주체의 동의) 및 제2호 (계약의 체결·이행)

### 1.3 유료 서비스 이용 시 수집 정보 (선택)

**결제 정보**
- Toss Payments를 통한 결제 승인 키 (orderId, paymentKey)
- 결제 금액, 결제 방법, 구독 플랜 정보

**법적 근거**: 개인정보 보호법 제15조 제1항 제2호 (계약의 체결·이행)

### 1.4 자동으로 수집되는 정보

**서비스 이용 기록**
- 접속 IP 주소, 쿠키, 접속 로그, 서비스 이용 기록, 페이지뷰
- 기기 정보 (브라우저 종류, OS, 화면 크기)

**법적 근거**: 개인정보 보호법 제15조 제1항 제6호 (공익적 목적) - 서비스 품질 개선 및 보안

### 1.5 민감정보 처리 (사업자등록번호)

**사업자등록번호 암호화 처리**
- **암호화**: AES-256-GCM (NIST 승인 알고리즘)
- **해싱**: SHA-256 (검색 및 중복 확인용)
- **저장**: 암호화된 형태로만 데이터베이스 저장

**법적 근거**: 개인정보 보호법 제24조 제1항 (고유식별정보의 암호화 저장)

## 3. 개인정보의 보유 및 이용 기간

Connect는 법령에 따른 개인정보 보유·이용기간 또는 정보주체로부터 개인정보를 수집 시 동의받은 개인정보 보유·이용기간 내에서 개인정보를 처리·보유합니다.

### 3.1 회원 탈퇴 시 즉시 파기

**파기 대상 정보**
- 회원 계정 정보 (이름, 이메일, OAuth 연동 정보)
- 조직 프로필 정보
- 구독 정보 (활성 구독이 없는 경우)

**파기 시점**: 회원 탈퇴 요청 후 **즉시 파기** (시스템 자동 처리)

**법적 근거**: 개인정보 보호법 제21조 제1항 (보유기간 경과 시 파기 의무)

### 3.2 회원 탈퇴 후 30일 보관 후 파기

**보관 대상 정보** (비식별화 처리)
- 매칭 데이터 (성공률, 추천 과제 수)
- 피드백 데이터 (서비스 개선용)
- 거래 기록 (분쟁 해결용)

**보관 기간**: 회원 탈퇴 후 **30일**

**법적 근거**: 전자상거래법 제6조 (거래기록 보관 의무)

### 3.3 법령에 따른 보관 의무

**보관 대상 정보 및 기간**

| 정보 항목 | 보관 기간 | 법적 근거 |
|----------|---------|---------|
| 계약 또는 청약철회 등에 관한 기록 | 5년 | 전자상거래법 제6조 |
| 대금결제 및 재화 등의 공급에 관한 기록 | 5년 | 전자상거래법 제6조 |
| 소비자의 불만 또는 분쟁처리에 관한 기록 | 3년 | 전자상거래법 제6조 |
| 표시·광고에 관한 기록 | 6개월 | 전자상거래법 제6조 |

## 4. 개인정보의 제3자 제공

Connect는 원칙적으로 정보주체의 개인정보를 제3자에게 제공하지 않습니다. 다만, 다음의 경우에는 예외로 합니다.

### 4.1 정보주체의 동의가 있는 경우

**제공받는 자**: 연구개발 과제 수행 파트너 (컨소시엄 구성원)

**제공 항목**: 조직명, 담당자 이름, 연락처, 이메일, 연구개발 역량 정보

**제공 목적**: 컨소시엄 구성 및 공동 연구개발 과제 수행

**보유 기간**: 프로젝트 종료 시까지 또는 제공 동의 철회 시까지

**법적 근거**: 개인정보 보호법 제17조 제1항 제1호 (정보주체의 동의)

### 4.2 법령에 특별한 규정이 있는 경우

**제공받는 자**: 법원, 검찰, 경찰, 한국인터넷진흥원 등 법률에 따라 권한이 부여된 기관

**제공 목적**: 법령상 의무 이행 (수사, 재판, 행정처분 등)

**법적 근거**: 개인정보 보호법 제18조 제2항 제7호 (법령상 의무 이행)

## 6. 개인정보의 파기 절차 및 방법

Connect는 원칙적으로 개인정보 처리목적이 달성된 경우에는 지체 없이 해당 개인정보를 파기합니다.

### 6.1 파기 절차

**회원 탈퇴 시 자동 파기 프로세스**

1. **탈퇴 요청 접수** (5단계 확인 프로세스)
   - 탈퇴 사유 입력 (선택)
   - 주의사항 확인 (체크박스 동의)
   - 구독 해지 처리 (활성 구독이 있는 경우)
   - 최종 확인 (비밀번호 또는 SMS 인증)

2. **데이터 파기 실행** (시스템 자동 처리)
   - PostgreSQL 트랜잭션: 회원 데이터 삭제
   - Redis 캐시: 세션 및 활성 사용자 데이터 삭제
   - 파일 스토리지: 업로드된 파일 삭제 (조직 로고 등)

3. **파기 확인**
   - 데이터 삭제 로그 생성 (감사 추적용)
   - 이메일 통지 (선택한 경우)

**파기 소요 시간**: 요청 후 **즉시** (시스템 자동 처리)

### 6.2 파기 방법

**전자적 파일 형태의 정보**
- 기술적 방법을 사용하여 복구 불가능하도록 영구 삭제
- PostgreSQL: `DELETE` 쿼리 + VACUUM 처리
- Redis: `DEL` 명령어로 즉시 삭제
- 파일 시스템: `rm -f` (복구 불가능)

**종이 문서 형태의 정보**
- 분쇄기를 이용한 파쇄 또는 소각 처리

### 6.3 보관 중인 정보의 격리 저장

**법령에 따라 보관해야 하는 정보**
- 별도의 데이터베이스에 격리 보관
- 접근 권한 제한 (DPO 및 법무팀만 접근 가능)
- 암호화 저장 (AES-256-GCM)
- 보관 기간 만료 시 자동 파기

## 7. 정보주체의 권리·의무 및 행사 방법

정보주체는 개인정보 보호법에 따라 다음과 같은 권리를 행사할 수 있습니다.

### 7.1 정보주체의 권리 (PIPA 제35조~제37조)

**1. 개인정보 열람 요구권 (제35조)**
- 자신의 개인정보를 열람하거나 사본을 요구할 수 있습니다.
- 대시보드 > 설정 > 내 정보 확인에서 언제든지 열람 가능

**2. 개인정보 정정·삭제 요구권 (제36조)**
- 개인정보에 오류가 있는 경우 정정을 요구할 수 있습니다.
- 대시보드 > 프로필 수정에서 직접 수정 가능

**3. 개인정보 처리정지 요구권 (제37조)**
- 개인정보의 처리 정지를 요구할 수 있습니다.
- 단, 법령에서 처리를 요구하는 경우 등은 예외

**4. 개인정보 이동권 (제35조의2)**
- CSV 형식으로 자신의 개인정보를 다운로드할 수 있습니다.
- 대시보드 > 설정 > 데이터 내보내기에서 다운로드 가능
- 제한: 1시간에 1회

### 7.2 권리 행사 방법

**온라인 자동 처리**
```
1. Connect 로그인
2. 대시보드 > 설정 메뉴 이동
3. 다음 기능 이용:
   - 내 정보 확인 (열람)
   - 프로필 수정 (정정)
   - 데이터 내보내기 (이동권)
   - 회원 탈퇴 (삭제)
```

**오프라인 서면 요청**
- 이메일: privacy@connectplt.kr
- 우편: (주소 미정)

**처리 기한**: 요청 접수 후 10일 이내 (개인정보 보호법 제35조 제5항)

### 7.3 법정대리인의 권리 행사

만 14세 미만 아동의 법정대리인은 아동의 개인정보에 대한 열람, 정정·삭제, 처리정지를 요구할 수 있습니다.

**참고**: Connect는 만 14세 이상만 회원가입 가능합니다.

### 7.4 정보주체의 의무

정보주체는 자신의 개인정보를 최신 상태로 유지해야 하며, 타인의 정보를 도용하거나 부정하게 사용하는 행위를 하지 않아야 합니다.

## 8. 개인정보 보호를 위한 기술적·관리적 보호대책

Connect는 개인정보 보호법 제29조에 따라 다음과 같은 기술적·관리적 보호대책을 시행하고 있습니다.

### 8.1 기술적 보호대책

**1. 개인정보 암호화 (PIPA 제24조 제3항)**
- **민감정보 암호화**: AES-256-GCM (사업자등록번호)
- **통신 구간 암호화**: HTTPS (TLS 1.3)
- **데이터베이스 암호화**: PostgreSQL Transparent Data Encryption (TDE)
- **파일 암호화**: 업로드된 파일 서버 측 암호화 (SSE-S3)

**2. 해싱 (검색 및 중복 확인)**
- **알고리즘**: SHA-256 (NIST FIPS 180-4)
- **적용 대상**: 사업자등록번호 (암호화 전 해싱 병행)

**3. 접근 통제**
- **역할 기반 접근 제어 (RBAC)**: USER, ADMIN, SUPER_ADMIN 역할 분리
- **세션 관리**: Redis 기반 세션 (30일 자동 만료)
- **API Rate Limiting**: Redis 기반 속도 제한 (예: 데이터 내보내기 1시간 1회)

**4. 로그 관리**
- **접속 로그 기록**: IP 주소, 접속 시간, 서비스 이용 기록
- **보관 기간**: 3개월 (통신비밀보호법 제15조의2)
- **로그 암호화**: 개인정보 포함 로그는 암호화 저장

**5. 보안 프로그램 설치 및 운영**
- **방화벽**: AWS Security Group 기반 접근 제어
- **침입 탐지**: CloudWatch를 통한 실시간 모니터링
- **취약점 점검**: 월 1회 정기 점검

### 8.2 관리적 보호대책

**1. 개인정보 보호 조직 구성**
- 개인정보 보호책임자 (DPO) 지정
- 개인정보 보호 담당 부서 운영

**2. 내부 관리계획 수립 및 시행**
- 개인정보 처리 직원의 최소화 및 교육
- 개인정보 취급자의 정기적인 교육 실시 (연 2회)
- 내부 관리계획 연 1회 검토 및 개정

**3. 개인정보 취급자 관리**
- 업무 인수인계 시 개인정보 보안 서약서 작성
- 퇴직 시 개인정보 반납 확인

**4. 물리적 보호대책**
- 개인정보 처리 시스템 접근 제한 (VPN 필수)
- 서버실 출입 통제 (클라우드 인프라는 AWS 관리)
